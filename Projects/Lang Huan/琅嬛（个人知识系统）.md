# 琅嬛（个人知识系统）

`琅嬛` 是一个面向个人学习与研究的知识工程项目。它的目标不是把所有笔记堆进一个搜索框，而是把 Obsidian 的人工结构、Zotero 的文献管理、RAG（检索增强生成）和 agent 工作流接成一个可维护的系统。

## Core Idea

这个项目把知识库分成三层：

1. **人工结构层**：`Home`、`Areas`、`Maps`、MOC 和概念节点仍然由人维护，负责长期导航与复习。
2. **检索辅助层**：本地 RAG 负责在大量 Markdown 笔记中定位上下文，返回文件、标题路径、片段和相关链接。
3. **Agent 执行层**：agent 使用 RAG 找到候选上下文后，必须打开源文件核对，再进行整理、改写或概念沉淀。

## Design Principles

- RAG 是上下文定位器，不是事实来源。
- 原始 PDF、图片、临时 raw/draft 和敏感配置不进入公共笔记与检索索引。
- Obsidian 的 `wikilink`、frontmatter 和 heading 结构是知识库的重要元数据，应被解析而不是当作普通文本丢弃。
- 公开版本只展示技术架构与可复用概念，不展示私人阅读内容、密钥、本地路径或运行日志。

## Current Baseline

当前公开版记录的是 RAG v1 基线：

- Dense vector retrieval（向量语义召回）
- BM25 sparse retrieval（关键词召回）
- Reciprocal Rank Fusion（RRF，倒数排名融合）
- Cross-Encoder Rerank（交叉编码器重排序）
- Heading-aware chunking（标题感知分块）
- Incremental indexing（增量索引）
- Local logs / optional observability（本地日志与可选可观测性）
- Explicit feedback loop（显式反馈闭环）
